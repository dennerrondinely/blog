### Real Example: Identifying Unnecessary Renders

With the React Profiler you'll see:

1. **UserList renders every time** state changes
2. **All UserCard components render** even if the specific user didn't change
3. **Total time** increases exponentially with the number of users

### Solution with memo()

```typescript

        <UserCard
          key={user.id}
          user={user}
          onSelect={handleSelect}
        />
      ))}
    </div>

```

Now with the React Profiler you'll see:

Lighthouse is a tool from **Google Chrome** that does a complete audit of your web application.

### What it does

Lighthouse measures:

- âš¡ **Performance** - Overall page speed
- â™¿ **Accessibility** - Accessibility for users with disabilities
- ğŸ“± **Best Practices** - Compliance with recommended practices
- ğŸ”’ **Security** - Security vulnerabilities
- ğŸ“Š **SEO** - Search engine optimization

### How to Use

1. Open Chrome DevTools (F12 or Cmd+Option+I)
2. Go to the **Lighthouse** tab
3. Select what you want to audit (can be Mobile or Desktop)
4. Click **Analyze page load**
5. Wait for the complete report

### Key Metrics

#### Core Web Vitals

Lighthouse focuses on three critical metrics:

- **LCP** (Largest Contentful Paint)
  - When the largest visible element appears
  - Target: `< 2.5s`

- **FID** (First Input Delay)
  - How long the page takes to respond to user input
  - Target: `< 100ms`

- **CLS** (Cumulative Layout Shift)
  - How much the layout moves while the page loads
  - Target: `< 0.1`

### STEP 1: Start with Lighthouse

Do a general audit:


**Look for:**

- Bad Core Web Vitals

- Large images

### STEP 2: Use React Profiler to Investigate

If Lighthouse pointed out "Unused JavaScript" or "Slow rendering":

1. Open React Profiler
2. Record a common interaction (scroll, button click)
3. Look for:
   - Renders that take too long
   - Re-renders in cascade
   - Components that render for no reason

### STEP 3: Implement Optimizations

Based on what you found:


- [ ] Use `React.memo()` for components receiving many props

- [ ] Optimize images (use WebP, lazy load)
- [ ] Implement code splitting
- [ ] Use CDN for static assets
- [ ] Implement service workers for caching
